Patching generally means applying an update or fix to existing software, hardware, or systems to correct problems, improve security, or add minor functionality.

R&D (Reasearch & Development team) will fix the issues (bugs) & provide patches. So, PATCH is a fix of bugs.
If ower DB is down and no one can fix the issues then we have to ask Oracle to resolve the issue. And to approach Oracle we need the help of METALINK website i.e. MOS(My Oracle Support) support.oracle.com.
MOS is a ticketing tool - In MOS there is SR(Service Request) & we have to raise SR, where we will mention all the details, issues & logs and thses SR will be tracked by the MOS based on the priority. If the business is completely in out of situation then P1 ticket its critical(Customer has to respond within 15 min), P2 ticket High(Certain slowness), P3 ticket Medium & P4 ticket Low.
Now Oracle will go through the SR and come up with solutions & if still we are unable to fix then they will be on call (zoom link or etc) and fix together, they will provide all the steps to fix. So, if it's a bug Oracle will provide a patch to fix which we will apply to fix the BUG.
